Note technique sur les URI définies dans l'espace de noms http://rdf.insee.fr/geo/

Deux types d'URI (Uniform Resource Identifier) sont définies dans cet espace de noms.

  • Des URI de type "URL" permettant d'accéder à des "ressources physiques", par exemple :

- La présente page HTML :       http://rdf.insee.fr/geo/note-uri.html

- Le fichier RDF "Ontologie":    http://rdf.insee.fr/geo/ontologie-geo-2006.rdf

- Le fichier RDF "Régions":      http://rdf.insee.fr/geo/regions-2003.rdf


  • Des URI identifiant des "objets abstraits" de l'ontologie :

- Des classes, comme par exemple :                          http://rdf.insee.fr/geo/Commune

- Des propriétés, comme par exemple :                      http://rdf.insee.fr/geo/chef-lieu

- Des instances (individus), comme par exemple :    http://rdf.insee.fr/geo/DEP_05


Aucune "ressource physique" ne correspond a priori à ce deuxième type d'URI. Une pratique proposée par une récente note du W3C [1] est que dans ce cas, une telle URI  permette d'accéder néanmoins à une ressource physique, de nature définie par le dialogue client-serveur, à savoir :

  • Une description formelle (c'est-à-dire un fichier RDF) pour une requête de type "HTTP GET - Accept application/rdf+xml".
  • Une description informelle (c'est-à-dire un fichier HTML) pour une requête de type "HTTP GET - Accept text/html".

Dans le cas présent, cette pratique a été jugée trop lourde à mettre en oeuvre à un niveau suffisant de détail, au moins pour les instances. Par exemple les éléments de description de l'entité identifiée par l'URI http://rdf.insee.fr/geo/DEP_05 sont distribués dans quatre fichiers différents : le fichier des régions, le fichier des départements, et les deux fichiers arrondissements-05 et cantons-05. Pour que l'URI en question permette d'accéder à une description complète de l'entité, il faudrait par exemple qu'elle déclenche une requête sur une base de données RDF, qui ramènerait un fichier contenant tous les triplets où cette entité est présente. Une autre solution serait que tous ces fichiers soient pré-publiés, ce qui voudrait dire un fichier de description RDF séparé pour chaque URI. Par ailleurs à chacune de ces URI devrait aussi correspondre une page de description HTML "à usage humain" pour les requêtes de type HTTP GET Accept text/html.

Ces choix techniques sont envisageables à terme mais ne sont pas mis en oeuvre pour le moment par l'INSEE. Pour éviter les erreurs 404 liées à l'utilisation éventuelle de telles URI dans le protocole HTTP, elles sont toutes redirigées vers une page qui conduit à la présente page informative http://rdf.insee.fr/geo/note-uri.html.

[1] Best Practice Recipes for Publishing RDF Vocabularies http://www.w3.org/TR/swbp-vocab-pub/
